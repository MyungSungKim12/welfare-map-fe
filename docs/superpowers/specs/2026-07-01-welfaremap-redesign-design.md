# WelfareMap 전체 디자인 재설계 명세

작성일: 2026-07-01

## 1. 목표

현재 WelfareMap은 밝은 배경, 둥근 카드, 초록/남색 강조, 큰 히어로가 반복되어 AI 템플릿처럼 보이는 인상이 있다. 이번 재설계는 사용자가 매일 확인하는 공공서비스형 생활 대시보드로 전환하는 것을 목표로 한다.

핵심 방향은 `공공서비스형 생활 대시보드 + 조용한 AI 추천 레이어`다. 복지 정책 검색, 마감 확인, 주변 시설 탐색, 저장한 복지, 향후 AI 체크리스트를 하나의 신뢰도 높은 업무형 화면으로 묶는다.

## 2. 디자인 원칙

- 신뢰감: 과한 장식보다 명확한 정보 위계를 우선한다.
- 실사용성: 첫 화면에서 위치, 검색, 마감, 저장, 지도, 리스트 진입이 바로 가능해야 한다.
- 낮은 AI 과시: `AI` 문구는 핵심 기능 설명에만 쓰고, 화면 전체를 AI 제품처럼 꾸미지 않는다.
- 정보 밀도: 카드 사이 여백과 반복 배지를 줄이고, 정책명/대상/지역/마감일을 빠르게 비교할 수 있게 한다.
- 확장성: 로그인, 마이페이지, 상세, AI 추천, 신청 체크리스트 화면이 같은 디자인 언어로 확장되어야 한다.
- 접근성: 버튼 최소 높이 44px, 명확한 focus/hover, 충분한 대비, 모바일 텍스트 줄바꿈 안정성을 지킨다.

## 3. 비주얼 시스템

### 색상

기존의 초록/남색 중심 팔레트는 유지하되 채도를 낮추고, 화면 전체를 한 가지 색상으로 덮지 않는다.

- 배경: `#F7F8F5`, `#F1F4EF`
- 기본 표면: `#FFFFFF`, `#FBFCFA`
- 주요 텍스트: `#17201C`
- 보조 텍스트: `#68746E`
- 경계선: `#DDE3DD`
- 주요 액션: `#245B4F`
- 보조 액션: `#263B45`
- 정보 강조: `#6E7F5B`
- 마감/위험: `#B84A3A`
- 주의/준비: `#A7672B`

### 타이포그래피

- 폰트는 현재 `Noto Sans KR`을 유지한다.
- 히어로급 제목은 줄이고, 대시보드 화면에 맞는 실무형 크기 체계를 둔다.
- `h1`: 3.6rem 내외, 모바일 2.8rem
- 섹션 제목: 2.2rem-2.8rem
- 카드 제목: 1.6rem-1.9rem
- 본문: 1.35rem-1.55rem
- letter-spacing은 0을 유지한다.

### 형태

- 반복 카드 반경은 10px-14px 중심으로 줄인다.
- 큰 패널은 16px까지 허용한다.
- pill 배지는 정말 필요한 상태값에만 사용한다.
- 그림자보다 경계선과 배경 면 분리로 깊이를 만든다.
- 그라데이션, 유리효과, 장식성 배경은 제거하거나 매우 제한한다.

## 4. 메인 화면 정보 구조

### 4.1 Navbar

현재 로고, 위치, 저장, 알림, 로그인 구조는 유지한다. 다만 랜딩 페이지형 중앙 정렬 대신 서비스 콘솔 상단바처럼 정리한다.

- 좌측: WelfareMap 로고와 짧은 상태 라벨
- 중앙 또는 좌측 보조: 현재 위치 버튼
- 우측: 저장, 알림, 로그인
- 모바일: 위치 버튼은 두 번째 줄 전체 폭이 아니라 상단바 내부에서 압축 표시한다.

### 4.2 Hero를 Command Center로 전환

현재 큰 홍보형 히어로를 `오늘의 복지 현황` 패널로 바꾼다.

- 제목은 서비스 설명보다 현재 사용자의 상황을 보여준다.
- 검색창은 첫 화면의 핵심 입력으로 유지한다.
- 현재 위치 갱신은 보조 버튼으로 배치한다.
- 총 후보, 마감 후보, 신규 정책은 작은 메트릭 바 형태로 정리한다.
- 오른쪽 추천 패널은 큰 카드 2개 대신 `오늘 확인할 일` 리스트로 압축한다.

예상 문구:

- `인천 미추홀구 기준, 확인할 복지와 주변 기관`
- `마감, 신규, 저장한 항목을 한 화면에서 점검하세요.`

### 4.3 LifeStage

현재 6개 큰 카드 구조는 과하게 보인다. `빠른 필터 레일`로 전환한다.

- 아이콘은 유지하되 크기를 낮춘다.
- 한 줄 또는 두 줄의 세그먼트 버튼처럼 배치한다.
- 향후 클릭 시 실제 필터와 연결될 수 있게 active 상태 디자인을 포함한다.

### 4.4 NoticeBanner

현재 3개 공지 카드는 PopularList와 역할이 겹친다. `Action Queue`로 개편한다.

- 마감 임박, 신규, 신청 준비를 한 줄 알림 리스트로 구성한다.
- 각 항목은 심각도, 제목, 설명, 바로가기 액션을 가진다.
- 너무 큰 카드 3개 대신 스캔 가능한 업무 목록처럼 만든다.

### 4.5 PopularList

현재 카드형 추천 레일은 유지하되 시각적 무게를 줄인다.

- `추천 레이더` 표현은 `우선 확인할 지원`으로 변경한다.
- 탭은 작은 segmented control로 정리한다.
- 카드 대신 compact rank list 또는 얇은 카드 리스트를 사용한다.
- 향후 BE popular API와 연결될 때 조회/저장/클릭 점수를 자연스럽게 보여줄 수 있어야 한다.

### 4.6 SavedSection

저장한 복지는 사용자의 재방문 동기이므로 더 명확한 보관함 형태로 만든다.

- 저장 항목이 있으면 메인 작업영역 위에 `내 보관함`으로 노출한다.
- 저장 항목이 없으면 섹션 자체를 숨기는 현재 정책은 유지한다.
- 향후 로그인 후 Supabase sync 상태를 보여줄 공간을 남긴다.

### 4.7 Welfare Workspace

현재 가장 중요한 화면이다. `필터 + 지도 + 리스트`를 유지하되 업무형 레이아웃으로 정리한다.

- 좌측 필터는 sticky 유지, 단 카드 장식 축소
- 지도와 리스트는 동일한 높이와 정돈된 패널로 구성
- 리스트 패널을 더 넓게 두고, 지도는 주변 시설 확인용 보조 패널로 둔다
- 모바일에서는 필터 → 요약 → 리스트 → 지도 순서가 더 자연스럽다
- 향후 상세 drawer 또는 route 이동을 붙이기 쉬운 구조를 유지한다

### 4.8 Statistics

현재 어두운 대형 통계 섹션은 페이지 하단에서 갑자기 마케팅 블록처럼 보인다.

- `지역 데이터 요약` 또는 `조회 기준 요약`으로 톤을 낮춘다.
- 전체 dark band 대신 차분한 summary panel로 변경한다.
- 카테고리 분포는 막대 리스트로 유지하되, 과한 대비를 줄인다.

### 4.9 Footer

Footer는 서비스 신뢰 정보, 데이터 출처, 법적/개인정보 링크 중심으로 간결하게 둔다.

- 장식성 radial background 제거
- 데이터 출처와 서비스 상태를 명확히 표시

## 5. 향후 화면 확장 규칙

### 복지 상세 화면

- 상단: 정책명, 지역, 대상, 마감 상태
- 본문: 요약, 신청 조건, 지원 내용, 신청 방법
- 우측 또는 하단: AI 준비 체크리스트, 저장, 원문 링크
- 상세 화면은 마케팅 상세 페이지가 아니라 문서형 정책 뷰어로 설계한다.

### 로그인 / 회원가입

- 간단한 인증 진입 화면으로 구성한다.
- 카카오/네이버 OAuth 버튼을 명확하게 배치한다.
- 로그인 이점을 과장하지 않고 저장 동기화, 알림, 맞춤 추천만 설명한다.

### 마이페이지

- 프로필 조건, 저장한 복지, 최근 본 정책, 알림 설정, 위치 설정으로 구성한다.
- 큰 카드 나열보다 설정 패널과 리스트 중심으로 설계한다.

### AI 추천 화면

- `AI`를 전면 캐릭터처럼 쓰지 않는다.
- 추천 이유, 필요한 서류, 다음 행동을 구조화된 checklist로 보여준다.
- 근거 데이터와 원문 링크를 항상 함께 제공한다.

### 알림 / 마감 관리

- 마감 D-day, 저장 여부, 신청 상태를 표 또는 리스트로 보여준다.
- 이후 이메일/푸시 알림 설정과 연결한다.

## 6. 구현 범위

1차 구현은 현재 메인 화면의 전체 디자인 재설계에 집중한다.

- `globals.css` 디자인 토큰 재정의
- Navbar 스타일 정리
- Hero를 command center 구조로 변경
- LifeStage compact filter rail화
- NoticeBanner action queue화
- PopularList compact list화
- SavedSection 보관함 톤 정리
- Welfare workspace 레이아웃 재정리
- WelfareCard 정보 밀도 개선
- Statistics/Footer 톤 다운
- LocationModal 시각 톤 정리

기능 로직은 유지한다. 데이터 흐름, API, 캐시, 북마크 로직은 이번 리디자인의 대상이 아니다.

## 7. 검증 기준

- `npm run lint` 통과
- `npm run build` 통과
- 주요 반응형 폭에서 텍스트 겹침 없음
- 모바일에서 검색창, 위치 버튼, 필터, 카드 액션이 44px 이상 터치 영역 유지
- Lighthouse나 브라우저 콘솔 기준 치명적 오류 없음
- AI 템플릿처럼 보이는 요소 제거 확인
  - 과한 히어로 문구 제거
  - 반복적인 둥근 카드/큰 배지 축소
  - 장식성 그라데이션/유리효과 축소
  - 실제 업무형 정보 구조 강화

## 8. 제외 범위

- BE popular API 연동
- 로그인/마이페이지 실제 구현
- AI 추천 API 구현
- Supabase 북마크 동기화
- 디자인 시스템 패키지 도입
- shadcn/ui 전면 도입

이번 작업은 기존 styled-components 구조 안에서 화면 품질을 끌어올리는 리디자인으로 제한한다.
